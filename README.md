ShareIt — Сервис аренды вещей (Backend)
Микросервисная система для аренды вещей между пользователями. Проект реализует полный цикл взаимодействия: от добавления вещи владельцем до бронирования, подтверждения заявки и оставления комментариев. Архитектура разделена на Gateway (API шлюз) и Server (бизнес-логика и данные).
Технологический стек
Язык: Java 21
Фреймворк: Spring Boot 3.3.2
Архитектура: Микросервисы (Gateway + Server)
Взаимодействие сервисов: RestTemplate (синхронные HTTP-вызовы)
База данных: PostgreSQL (через Spring Data JPA / Hibernate)
Валидация: Jakarta Validation (@Valid, кастомные аннотации)
Логирование: SLF4J / Logback
Сборка: Maven / Gradle
Контейнеризация: Docker (запуск через docker-compose)
Архитектура решения
Проект состоит из двух основных модулей:
1. ShareIt Gateway
Единая точка входа для клиентов.
Принимает HTTP-запросы, выполняет валидацию входных данных (DTO).
Маршрутизирует запросы к соответствующим клиентам (ItemClient, BookingClient, UserClient, CommentClient).
Формирует ответы клиенту, преобразуя ответы от сервера.
Контроллеры: ItemController, BookingController, UserController, CommentController.
2. ShareIt Server
Реализует бизнес-логику приложения.
Управляет сущностями: User, Item, Booking, Comment.
Обеспечивает целостность данных при создании бронирований и отзывов.
Обрабатывает исключения через GlobalExceptionHandler.
Основные функции:
CRUD операций для пользователей и вещей.
Логика бронирования (статусы: WAITING, APPROVED, REJECTED, CANCELED).
Проверка прав доступа (владелец вещи vs арендатор).
Создание комментариев только после завершенного бронирования.
Ключевой функционал
Пользователи (Users)
Регистрация и управление профилем.
Атрибуты: id, name, email (уникальный).
Вещи (Items)
Добавление вещей владельцем с указанием названия, описания и доступности.
Поиск вещей по тексту (полнотекстовый поиск по названию и описанию).
Просмотр статистики просмотров вещей.
Бронирования (Bookings)
Создание заявки на бронирование с указанием даты начала и конца.
Подтверждение или отклонение брони владельцем вещи.
Фильтрация бронирований по статусу и пользователю.
Автоматическая проверка конфликтов дат при создании.
Комментарии (Comments)
Добавление отзыва к вещи.
Бизнес-правило: Комментарий можно оставить только после успешного завершения бронирования этой вещи данным пользователем.

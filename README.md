# ShareIt — Сервис аренды вещей (Backend)

Распределенная система для организации шеринга вещей между пользователями. Проект реализует микросервисную архитектуру, разделяя точку входа (Gateway) и бизнес-логику с хранением данных (Server). Система обеспечивает полный цикл операций: регистрация пользователей, управление инвентарем, бронирование с проверкой доступности и модерация отзывов.

## Технологический стек

*   **Язык:** Java 21
*   **Фреймворк:** Spring Boot 3.3.2
*   **Архитектура:** Микросервисы (REST API)
    *   **ShareIt Gateway:** Маршрутизация запросов, валидация DTO, агрегация ответов.
    *   **ShareIt Server:** Бизнес-логика, транзакционное управление, работа с БД.
*   **Взаимодействие сервисов:** `RestTemplate` (синхронные HTTP-вызовы внутри кластера).
*   **База данных:** PostgreSQL (через Spring Data JPA / Hibernate).
*   **Валидация:** Jakarta Validation (`@Valid`, кастомные аннотации, группы валидации).
*   **Тестирование:** JUnit 5, Mockito, AssertJ.
*   **Сборка и запуск:** Maven, Docker, Docker Compose.

## Архитектурное решение

Проект разделен на два независимых модуля:

### 1. ShareIt Gateway
Единая точка входа для внешних клиентов.
*   Принимает HTTP-запросы и проводит первичную валидацию входных данных.
*   Транслирует запросы к соответствующим клиентам серверной части (`UserClient`, `ItemClient`, `BookingClient`, `CommentClient`).
*   Обрабатывает ответы от сервера и формирует финальный JSON для клиента.
*   Не содержит прямой логики работы с базой данных.

### 2. ShareIt Server
Ядро системы, отвечающее за состояние данных и бизнес-правила.
*   Управляет сущностями: `User`, `Item`, `Booking`, `Comment`.
*   Реализует CRUD-операции и сложные сценарии взаимодействия (бронирование, подтверждение, отказ).
*   Обеспечивает целостность данных через транзакции (`@Transactional`).
*   Реализует глобальную обработку исключений (`GlobalExceptionHandler`) для возврата корректных HTTP-кодов.

## Ключевой функционал и бизнес-правила

### Пользователи (Users)
*   Регистрация и обновление профиля.
*   Уникальность поля `email` гарантируется на уровне базы данных и сервиса.
*   Атрибуты: `id`, `name`, `email`.

### Вещи (Items)
*   Создание вещей владельцем с указанием статуса доступности (`available`).
*   Полнотекстовый поиск по названию и описанию.
*   Получение списка вещей владельца с информацией о последнем и следующем бронированиях.
*   Учет статистики просмотров.

### Бронирования (Bookings)
*   Создание заявки с интервалом времени (`start`, `end`).
*   Статусная модель: `WAITING` → `APPROVED` / `REJECTED` / `CANCELED`.
*   **Ограничения:**
    *   Нельзя забронировать вещь, если она недоступна (`available = false`).
    *   Нельзя создать пересекающееся бронирование для одной вещи.
    *   Владелец вещи не может забронировать свою же вещь.
*   Подтверждение или отклонение брони доступно только владельцу вещи.

### Комментарии (Comments)
*   Добавление текстового отзыва к вещи.
*   **Критическое бизнес-правило:** Комментарий может быть создан только пользователем, у которого есть завершенное (`status = APPROVED` и дата окончания прошла) бронирование данной вещи. Попытка оставить комментарий без истории успешной аренды возвращает ошибку `400 Bad Request`.
